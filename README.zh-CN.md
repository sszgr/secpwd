# secpwd

[English](./README.md) | **中文**

一个轻量的前端密码派生工具。你可以通过“主密码 + 辅助密码”组合，或直接输入 `SecKey` 来生成强度较高且可复制的密码，适合以纯静态页面方式自行部署。

## 功能特性

- 纯静态页面，无需后端
- 支持“双密码”模式和“安全 Key”模式
- 支持在密码生成时混入自定义特殊符号
- 支持调整输出密码长度
- 支持一键复制生成的 `Key` 和密码
- 响应式布局，适合移动端访问

## 工作原理

当前实现完全在浏览器内完成密码派生：

1. 对主密码和辅助密码分别进行 MD5 处理
2. 将中间值组合后经过 AES，再用 SHA256 生成 `SecKey`
3. 基于该 `SecKey` 派生最终密码
4. 界面会展示一个主结果和若干备选结果，便于直接使用

`sec.js` 中也保留了远程 API 的占位逻辑，如果你有需要，可以替换为自己的后端生成流程。

## 使用说明

### 双密码模式

- 主密码：输入一个只有你自己知道且能稳定记住的密码
- 辅助密码：输入和目标站点或用途相关的标识，例如 `github`、`mail` 或 `@example.com`
- 页面会自动生成 `SecKey` 和对应派生密码

### 安全 Key 模式

- 直接输入一个 `SHA256` 值作为 `SecKey`
- 页面会基于该 key 派生密码

### 特殊符号

- 开启后可在密码生成中包含特殊符号
- 可以自定义允许使用的符号，例如 `!@#$%^&*`

### 密码长度

- 通过页面中的滑块控制展示出来的密码长度
- 支持范围：`6` 到 `32`

## 安全提示

- 当前仓库采用前端本地派生流程，因此核心逻辑在浏览器中可见
- 如果你需要更强的控制能力，可以将生成流程迁移到自建后端 API
- 不要在不可信的第三方部署页面中输入真实主密码
